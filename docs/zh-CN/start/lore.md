# Longxia 分支说明

Longxia 是基于 OpenClaw 的受限分支，目标是保留本地网关、CLI、控制界面和已审查能力，同时收紧默认网络边界。

## 主要变化

- 移除了依赖 Telegram、Discord、WhatsApp 的公开消息渠道默认路径
- 移除了运行时插件安装和更新能力
- 安装链路改为优先使用仓库内脚本、本地文档和国内可访问镜像
- 默认推荐本地部署或自控基础设施，不再鼓励依赖第三方社区入口

## 为什么这样做

这个分支的重点不是“功能越多越好”，而是：

- 降低对外网公共服务的耦合
- 降低运行时动态下载与扩展带来的审计成本
- 让默认安装路径在国内网络环境下更稳定、更可控

## 与上游的关系

Longxia 仍然继承了大量 OpenClaw 的结构和能力，但这里的文档与产品边界已经不同：

- 用户可见说明优先指向仓库内的 `./docs/...`
- 默认流程强调本地、自控、可审计
- 旧的上游名称、历史链接或发布记录，可能仍出现在注释、变更记录或历史资料里

## 文档编写规则

为 Longxia 新增说明时，遵循这几点：

- 优先写本地路径，不把用户引导到公共社区平台
- 外部服务要么明确标注为例外，要么默认视为不推荐
- 如果某功能依赖未审查的公网下载或第三方社群，应先从默认流程中移除

## 当前默认立场

凡是依赖公共社交平台、运行时外部下载、或难以在国内稳定访问的公共入口，都不属于 Longxia 的默认支持范围，除非后续以“国内可达 + 可自控”的方式重新接入。
