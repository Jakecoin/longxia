# Fly.io 部署

Longxia 不再提供面向 Fly.io 的推荐部署路径。

原因很简单：这个分支已经移除外部聊天渠道接入、运行时插件安装以及依赖第三方公网模板的快速上线方式。上游这类平台文档通常会顺带引导：

- 配置 Telegram 或 Discord 令牌
- 暴露公开访问入口
- 在服务器上继续做外部渠道配对

这些都不再是 Longxia 的支持范围。

## 当前建议

- 在你自己可控的环境中部署 Longxia。
- 只开放 Longxia 自有服务所需的最小边界。
- 不把第三方平台模板当作默认产品入口。

如果后续需要整理新的部署方案，应以 Longxia 自己维护的部署规范为准，而不是复用上游的 Fly.io 公网模板。
