# 在 Railway 上部署

Longxia 不再提供 Railway 一键模板部署说明。

上游这类页面的目标是快速暴露一个公网可访问实例，并继续接入 Telegram、Discord、WhatsApp 或其他外部渠道。Longxia 当前的产品边界与之相反：

- 不引导到外部聊天平台
- 不提供运行时插件安装
- 不把第三方托管模板作为默认上线入口

## 当前处理方式

如果你只是要验证功能，请优先使用本地开发环境与内置界面。

如果你需要托管部署，请在你自己的受控环境中完成，并确保：

- 只保留 Longxia 自己需要的服务边界
- 不复用上游的外联渠道变量与说明
- 不把第三方平台页面中的公网暴露步骤直接照搬到生产

后续若需要正式的托管文档，应单独为 Longxia 编写，而不是沿用上游 Railway 模板。
